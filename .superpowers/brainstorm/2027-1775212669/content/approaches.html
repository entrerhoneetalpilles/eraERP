Approches pour le Bloc 1 — Fondations + CRM

Architecture validée : Turborepo + NestJS + Next.js. Reste à choisir les outils clés.

ORM & Base de données
A

Prisma ⭐ Recommandé

Schema déclaratif, migrations auto, client TypeScript typé à 100%. Standard de l'industrie Next.js/NestJS. Multi-tenant via schema PostgreSQL ou colonne tenantId.

B

Drizzle ORM

Plus performant, SQL-first, très léger. Moins mature pour les migrations complexes. Bon choix si performance critique dès le départ.

Authentification
C

Better-Auth + JWT ⭐ Recommandé

Lib auth moderne pour Next.js + NestJS. TOTP/MFA natif, sessions + JWT, OAuth providers (Google/Apple). Hébergé en France possible.

D

NextAuth v5 (Auth.js)

Très répandu, excellent avec Next.js mais plus complexe à intégrer côté NestJS. MFA nécessite un plugin tiers.

Fetching & State côté front
E

TanStack Query v5 ⭐ Recommandé

Cache, invalidation, optimistic updates. Parfait avec une API REST. Standard pour les back-offices complexes.

Ma stack finale recommandée : Prisma + Better-Auth + TanStack Query. Réponds "ok" dans le terminal pour valider, ou précise si tu veux changer quelque chose.
